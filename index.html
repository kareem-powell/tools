How to bring this suite into your own environment

Every tool in this suite is plain HTML — no build step, no framework, no dependencies. If you're on the OM or Sales Finance team and want to adapt these for yourself, follow these five steps. You'll have a live version running in under ten minutes.

Step 01
Fork the repository on GitHub

Forking creates your own copy of the repo under your GitHub account. Every tool, every file, yours to edit — without touching the original.

github.com/kareem-powell/tools/fork ↗
Step 02
Clone it to your local machine

Replace YOUR-USERNAME with your GitHub handle. This downloads the full repo to your computer.

git clone https://github.com/YOUR-USERNAME/tools.git
Step 03
Open any file directly in your browser

No server required. Navigate to the cloned folder and open bookings.html, workbud.html, or index.html in Chrome. Everything runs client-side.

Tip — on Mac: open bookings.html from Terminal opens it immediately.
Step 04
Adapt the tool cards and content to your role

Each tool card in index.html is a plain <a> tag. Update the title, description, icon, and href to match your own links and workflow. The Work Bud password is set in workbud.html — search for CORRECT_PASSWORD and replace it with your own.

The Booking Checker checklist logic lives in a single JS function — swap in the checklist items relevant to your deal types.
Step 05
Deploy to GitHub Pages — your suite goes live

In your forked repo on GitHub, go to Settings → Pages. Under "Branch," select main and save. Within about 60 seconds, your suite will be live at:

https://YOUR-USERNAME.github.io/tools/
Push any update to main and GitHub Pages redeploys automatically — usually within 30 seconds.
